HUAWEI
THESIS

竞争重心正在从“补全代码”转向“完成工程任务”

01
阶段迁移

2022-2024 年是代码助手阶段，2025 年进入 vibe coding，2026 年开始形成 Agentic Engineering。工具不再只给建议，而是具备规划、推理、环境感知、工具调用和闭环验证能力。

02
阵营分化

市场分成 AI 原生 IDE、IDE 插件扩展、终端/CLI 编程代理三大阵营。入口不同，决定上下文来源、执行权限、团队集成方式和商业化节奏。

03
护城河变化

模型分数会继续收敛，真正拉开差距的是上下文工程、长程记忆、多智能体协作、协议标准、安全沙盒、可验证交付和企业权限审计。

04
三年推演

2027 年视觉到代码和多模型编排进入主流，2028 年多代理协作成为标配，2029 年竞争进入开放生态、企业治理和行业解决方案的综合较量。

产品路线分化
HUAWEI
OPERATING MODEL

四家竞争的不是同一种工具，而是四种工程组织方式

新增报告强调：编程智能体的竞争焦点已经从生成速度，转向任务成功率、架构理解深度、环境掌控力和人与智能体的协作结构。

Claude Code：终端原生精确主义

通过 Unix 风格工作流、CLAUDE.md、自主记忆和 Agent Teams，把智能体嵌入已有工程命令体系，强调对架构与代码库的精确干预。

Codex：全自动任务委托中枢

围绕 `/goal` 把自然语言目标拆成计划、命令、代码、测试和修复循环，开发者从实现者转向工程经理式的目标定义与验收。

OpenCode：开放生态守望者

用 Plan/Build 双智能体、ACP、多编辑器和 75+ 模型提供商构建反锁定路线，核心价值是选择权、可扩展性和成本控制。

Antigravity：任务指挥部

以智能体编排为中心，支持动态子智能体、异步执行、计划任务和 Google 云生态串联，试图把 IDE 改造成控制中心。

上下文工程
HUAWEI
CONTEXT ENGINEERING

长上下文不是终点，真正的胜负在“如何记住与如何遗忘”

新增报告指出：单纯扩大到 1M+ tokens 会带来 KV 缓存压力、注意力衰减和推理成本。主流智能体正在用适应性压缩、工具结果清理和分层记忆维持长程任务质量。

Adaptive Compaction

对话接近限制时，把历史对话、工具结果和中间推理压缩成高保真摘要，再开启新会话。

Tool-result Clearing

冗长构建日志和可再生成输出被替换为占位摘要，回收空间但保留关键推理依据。

Hierarchical Memory

短期记忆承载当前任务流，长期记忆沉淀用户偏好、项目事实和架构模式。

Project Constitution

Claude 的 CLAUDE.md 代表一种团队规范外化方式，防止智能体长期运行中的指令偏移。

协议之战
HUAWEI
MCP VS ACP

竞争已经进入协议层：谁定义连接方式，谁定义工具边界

新增报告把 MCP 与 ACP 视为下一阶段核心战场。MCP 解决“智能体如何连接外部工具”，ACP 解决“编辑器如何驱动和切换智能体”。两者共同决定开发者能否摆脱平台锁定。

MODEL CONTEXT PROTOCOL
MCP：工具连接层

由 Anthropic 主导，侧重让智能体以统一方式连接外部工具、数据源和企业系统。报告给出的信号是截至 2026 年下载量达 9700 万次，说明工具服务器生态已经形成规模化网络效应。

关键价值

一次编写工具服务器，可在多种支持 MCP 的智能体中复用。

竞争含义

工具生态越丰富，Claude 系产品越容易成为企业上下文入口。

AGENT CLIENT PROTOCOL
ACP：编辑器驱动层

由 JetBrains 和 Zed 联合发起，侧重让编辑器用统一协议驱动不同后端智能体。它解决 1:1 耦合问题，使开发者能够在同一界面下从 Claude Code 切换到 Codex 或 OpenCode。

关键价值

前端体验与后端模型解耦，降低用户迁移和试错成本。

竞争含义

OpenCode 等开放工具可借 ACP 进入多编辑器生态，削弱单一 IDE 锁定。

风险与治理
HUAWEI
RISK MANAGEMENT

代理越深入工程流程，安全与信任越成为采用前提

安全正确性不足

新增报告提到 Agent Security League 中智能体漏洞修复最高得分仅 23.5%，说明功能正确不等于安全正确。硬编码密钥、越权、注入和认证边界漂移仍是高风险区。

自主权限扩大

GTG-2002 一类攻击说明，智能体可被诱导攻击内部系统。企业必须把提示词限制、工具参数校验、运行时审批和沙盒隔离作为默认能力，而不是上线后的补丁。

平台锁定与合规压力

Claude-only、Gemini-only、OpenAI-only 或 Google 强绑定会提高迁移成本。企业需要多模型抽象、配置迁移、审计日志、隐私边界和知识产权保护机制。

Prompt Guardrails

在系统提示中定义禁止行为与团队安全边界。

Schema Gating

用 Plan/Build 等架构隔离限制危险工具访问。

Runtime Approval

删除文件、改 Git 历史、生产访问必须人工批准。

Lifecycle Hooks

行动前后插入审计脚本，形成可追溯治理链。

HUAWEI
CHALLENGER STRATEGY

后来者不要复制巨头
要选择非对称战场超越

面对 Claude Code、Codex、OpenCode、Antigravity 的先发优势，后来竞争者不应正面拼模型榜单，而要在工作流楔子、协议开放、可信治理和行业深度上建立可迁移优势。

01 先选楔子：避开通用编码红海
  • 不要从“更好的 Copilot / Claude Code”切入，优先选择高痛点、低替换阻力的场景：遗留系统迁移、CI 修复、安全修复、数据工程、DevOps/IaC。
  • 把卖点从“会写代码”改成“能完成一个可验收任务”，用测试通过率、交付周期、缺陷回滚率和人工节省时间做证据。
  • 先服务一个角色或流程，而不是一次性做通用 IDE：例如安全工程师、SRE、测试平台、企业代码迁移团队。
02 再建迁移能力：用开放协议反锁定
  • 默认支持 MCP / ACP / LSP / GitHub / GitLab / Jira / Slack，把自己做成可嵌入的执行层，而不是封闭入口。
  • 提供从 Claude Code、Codex、Cursor、Gemini CLI 迁移的规则、记忆、命令和任务模板导入能力，降低团队试用成本。
  • 坚持多模型路由和 BYOK：按任务自动选择快模型、深推理模型、本地模型或企业私有模型，避免被单一模型成本卡死。
03 最后超越：把信任做成护城河
  • 把权限审批、沙盒、审计、成本上限、密钥防护、自动化安全审查链做成产品默认能力，而不是企业版附加项。
  • 建立“任务成功率 + 安全正确率 + 成本/PR + 可回滚性”的复合指标，用企业可采购的语言证明价值。
  • 当巨头卖通用能力时，后来者要卖可落地的 Agentic SDLC 套件：需求到 PR、测试、安全、发布、运维闭环。
